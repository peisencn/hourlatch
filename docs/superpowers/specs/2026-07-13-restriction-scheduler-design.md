# 定时限制与临时放行功能设计

## 背景

本项目用于在每天指定时间段内提醒并约束电脑使用。进入限制时间段后，程序执行锁屏或休眠；如果用户在限制时间段内重新解锁或唤醒电脑，程序再次进入限制流程。遇到紧急情况时，用户可以输入独立密码获得有时限的临时放行。

该功能定位为自我提醒和轻度约束，不试图阻止拥有本机管理员权限的用户停止、卸载或绕过程序。

## 基础项目选择

项目基于 `lukaslangrock/ShutdownTimerClassic` 的 MIT 许可代码二次开发。

计划复用：

- WinForms 桌面界面和系统托盘基础
- 锁屏、睡眠等 Windows 电源操作
- 倒计时显示和通知交互
- 现有密码保护界面的部分交互思路

不沿用其单次倒计时作为核心模型。新的核心由每日时间段策略和状态机驱动。

`mirzaevolution/Vx-Shutdown-Timer` 仅作为 Windows Service 和系统操作的参考，不继承其旧版 `.NET Framework 4.5.2` 技术栈。MVP 不实现 Windows Service。

## 目标

- 支持一个每天重复的限制时间段。
- 时间段允许跨午夜，例如 `23:00-07:00`。
- 限制动作可选择锁屏或休眠。
- 在进入时间段、会话解锁和睡眠恢复后重新评估策略。
- 支持密码临时放行，并允许多次重新申请放行。
- 支持固定时长、自定义时长以及放行至当前时间段结束。
- 程序重启后恢复仍然有效的临时放行。
- 登录 Windows 后自动启动并常驻系统托盘。
- 避免配置错误造成无法操作的锁屏或休眠循环。

## 非目标

- 不接管 Windows 登录或安全桌面密码。
- 不阻止管理员停止进程、卸载程序或修改配置文件。
- MVP 不支持多个时间段、按星期设置或节假日规则。
- MVP 不实现 Windows Service、远程管理或家长控制平台。
- 不修改 Windows 全局电源计划。

## 架构

### TrayApplication

应用入口、设置窗口、系统托盘、限制提示窗口和通知展示。应用登录后自动启动，并在后台保持运行。

### SchedulePolicy

负责计算当前时间是否处于限制窗口，并返回当前窗口的开始和结束时间。计算必须正确处理普通窗口和跨午夜窗口。

### RestrictionController

系统总控和状态机。所有定时器、解锁、恢复、配置变化和系统时间变化事件必须进入该组件，由它决定是否提醒、执行限制、保持放行或结束当前窗口。

### SystemEventMonitor

监听以下信号：

- 应用启动
- Windows 会话解锁
- 系统从睡眠状态恢复
- 系统时间或时区变化
- 周期性校验计时器

事件只负责发出信号，不直接执行锁屏或休眠。

### PowerActionExecutor

封装锁屏和休眠操作，返回明确的成功或失败结果，并记录可诊断错误。该组件复用或改造基础项目中的 Windows 系统操作代码。

### OverrideManager

负责密码验证、临时放行创建、到期判断和持久化。每次密码验证成功后均可创建新的放行，不限制同一窗口内的使用次数。

### SettingsStore

负责读取、校验和原子写入配置。配置损坏或验证失败时，自动动作保持禁用，并向用户显示修复提示。

## 状态模型

`RestrictionController` 使用以下状态：

- `Disabled`：限制功能未启用或配置无效。
- `OutsideWindow`：当前不在限制时间段。
- `Warning`：即将执行限制动作，正在显示提醒倒计时。
- `Restricted`：已执行锁屏或休眠，等待下一次系统事件。
- `OverrideActive`：密码放行仍然有效。
- `ActionFailed`：系统动作执行失败，等待重试或用户处理。

状态转换必须是幂等的。多个相同系统事件不能创建多个提示窗口或连续执行多次休眠。

## 核心流程

### 进入限制时间段

1. 周期性校验发现当前进入限制时间段。
2. 如果不存在有效放行，则进入 `Warning`。
3. 显示默认 60 秒的执行前提醒。
4. 用户未获得放行时，执行锁屏或休眠。
5. 执行成功后进入 `Restricted`。

### 解锁或睡眠恢复

1. `SystemEventMonitor` 发出解锁或恢复信号。
2. `RestrictionController` 重新计算当前策略。
3. 如果仍在限制时间段且没有有效放行，显示限制提示窗口。
4. 用户未操作或验证失败时，提醒倒计时结束后重新执行限制动作。
5. 睡眠恢复但会话尚未解锁时，不立即反复休眠；等待会话解锁后再提供放行机会。

### 临时放行

密码正确后，用户可以选择：

- 15 分钟
- 30 分钟
- 1 小时
- 自定义时长
- 直到当前限制窗口结束

放行保存为绝对结束时间。到期时如果仍处于同一限制窗口，则重新进入 `Warning`。再次输入密码可以创建新的放行。

“直到窗口结束”的放行绑定当前窗口结束时间，不会延续到下一天的新窗口。

### 应用重新启动

1. 加载配置和持久化放行状态。
2. 校验放行是否仍属于当前窗口且尚未到期。
3. 有效时恢复 `OverrideActive`。
4. 无效时删除放行并按当前时间重新评估限制状态。
5. 如果启动时已在限制时间段，先显示提醒，不立即无提示休眠。

## 密码与安全

- 使用独立的应用密码，不尝试验证或替代 Windows 密码。
- 只保存盐值、PBKDF2 哈希、迭代次数和算法版本，不保存明文密码。
- 密码验证使用固定时间比较。
- 连续错误输入触发短暂冷却，但不做长期账户封禁。
- 修改密码、关闭限制功能、暂停本次限制和受保护退出时要求现有密码。
- 当前安全模型面向自我约束，不承诺抵御本机管理员绕过。

## 配置模型

配置至少包含：

- 功能是否启用
- 每日开始时间
- 每日结束时间
- 限制动作：锁屏或休眠
- 执行前提醒秒数
- 默认放行时长
- 是否显示“直到窗口结束”选项
- 登录后自动启动
- 密码哈希参数
- 当前临时放行的窗口标识和结束时间

开始时间和结束时间完全相同时视为无效配置，不能启用限制。

## 用户界面

### 设置窗口

- 限制功能开关
- 开始和结束时间
- 锁屏或休眠选择
- 提醒时长
- 默认放行时长
- 是否允许放行至窗口结束
- 设置或修改密码
- 登录后自动启动
- 当前状态和下一次动作时间

### 限制提示窗口

- 当前限制时间段
- 即将执行的动作
- 剩余提醒时间
- 密码输入框
- 放行时长选择
- 临时放行按钮
- 立即执行按钮

### 托盘菜单

- 当前状态
- 打开设置
- 临时放行
- 提前结束放行
- 暂停本次限制
- 退出程序

“暂停本次限制”和受保护退出需要密码。暂停状态仅持续到当前限制窗口结束，不永久关闭功能。

## 自动启动

MVP 使用当前用户登录后自动启动方式，不安装 Windows Service。实现应封装为独立接口，以便后续替换为 Windows 计划任务守护模式。

如果程序被管理员主动结束，限制不再执行，符合当前轻度约束定位。

## 错误处理

- 配置使用临时文件加替换方式原子写入。
- 配置损坏时禁用自动动作并提示用户，不使用可能造成锁定的猜测值。
- 系统动作失败时记录本地日志并显示托盘通知。
- 相同系统事件需要去重，限制窗口同时只允许存在一个提示窗口。
- 系统时间或时区变化后立即重新计算状态。
- 休眠失败不能进入持续高频重试；使用有上限的退避重试。
- 未设置密码时禁止启用限制功能。

## 验收标准

- 用户可以配置并启用一个每日限制时间段。
- 进入时间段后程序按配置提醒并锁屏或休眠。
- 时间段内每次解锁或睡眠恢复都会重新评估限制。
- 正确密码可以按用户选择的时长临时放行。
- 放行到期后程序重新执行限制流程。
- 同一时间段内可以重复使用密码放行。
- 程序重启后有效放行不会丢失。
- 损坏配置或系统调用失败不会产生无限锁屏或休眠循环。
- 密码不会以明文形式写入磁盘。
